# R2Nova 功能描述文档

**版本**: v1.0.0
**用途**: Stitch UI 设计参考 - 仅功能描述
**项目**: R2Nova - Cloudflare R2 桌面客户端

---

## 1. 产品概述

R2Nova 是 Cloudflare R2 桌面客户端，提供直观的文件管理体验。

### 核心价值

- **极致轻量**: ~5MB 安装包
- **大文件专家**: 原生支持 50GB+ 断点续传
- **本土化**: 原生中文界面
- **多账号**: 一键切换多账号

### 技术栈

- 桌面框架: Tauri 2.0 + Rust
- 前端: React 19 + TypeScript
- UI 组件: shadcn/ui + Tailwind CSS
- 状态管理: Zustand
- S3 客户端: aws-sdk-s3 (Rust)

## 3. 用户流程

### 3.1 首次使用流程

1. 打开应用
2. 点击"添加账号"
3. 输入 R2 Endpoint、Access Key ID、Secret Access Key
4. 保存账号
5. 自动进入存储桶浏览页面
6. 选择 Bucket 开始使用

### 3.2 文件上传流程

1. 在存储桶浏览页面点击"上传"按钮
2. 选择本地文件
3. 任务自动添加到传输中心
4. 显示上传进度
5. 上传完成后文件出现在列表中
6. 传输中心显示完成状态

### 3.3 文件下载流程

1. 在文件列表中找到目标文件
2. 点击下载按钮
3. 选择保存位置
4. 任务添加到传输中心
5. 显示下载进度
6. 下载完成后文件保存到指定位置

### 3.4 账号切换流程

1. 点击导航中的账号管理
2. 查看账号列表
3. 点击目标账号
4. 确认切换（如有进行中的传输任务提示）
5. 切换到新账号的存储桶列表

### 3.5 查看传输任务流程

1. 点击导航中的传输中心（显示任务数量徽章）
2. 查看所有传输任务列表
3. 可按上传/下载筛选
4. 查看任务详情和进度
5. 清除已完成的任务

### 3.6 取消上传流程

1. 在传输中心或文件列表中找到正在进行的上传
2. 点击取消按钮
3. 确认取消
4. 中止多部分上传会话
5. 任务状态变为已取消/失败
6. 从列表中移除

## 4. 数据结构

### 4.1 账号 (Account)

```typescript
interface Account {
  id: string // 唯一标识
  name: string // 账号名称（用户自定义）
  endpoint: string // R2 Endpoint URL
  access_key_id: string // Access Key ID（部分显示）
  created_at: string // 创建时间
  is_active: boolean // 是否当前使用
}
```

### 4.2 Bucket (BucketInfo)

```typescript
interface BucketInfo {
  name: string // Bucket 名称
  creation_date?: string // 创建时间
  object_count?: number // 对象数量
  total_size?: number // 总存储大小
}
```

### 4.3 文件/对象 (ObjectInfo)

```typescript
interface ObjectInfo {
  key: string // 对象键（路径）
  size: number // 文件大小（字节）
  last_modified?: string // 最后修改时间
  etag?: string // ETag
  is_directory: boolean // 是否为文件夹

  // 前端展示用字段（多部分上传）
  is_uploading?: boolean // 是否正在上传
  upload_progress?: number // 上传进度（0-100）
  upload_speed?: number // 上传速度（MB/s）
  upload_status?: string // 上传状态
  upload_id?: string // 多部分上传 ID
}
```

### 4.4 传输任务 (TransferTask)

```typescript
interface TransferTask {
  id: string // 任务 ID
  type: 'upload' | 'download' // 传输类型
  status: 'pending' | 'in_progress' | 'completed' | 'failed' // 状态
  filename: string // 文件名
  bucket: string // 存储桶名称
  key: string // 文件路径
  bytesTotal: number // 总大小
  bytesTransferred: number // 已传输大小
  speedMbps: number // 传输速度（MB/s）
}
```

### 4.5 多部分上传信息 (MultipartUploadInfo)

```typescript
interface MultipartUploadInfo {
  key: string // 文件路径
  upload_id: string // 多部分上传会话 ID
  initiated?: string // 开始时间
  storage_class?: string // 存储类别
}
```

## 6. 技术约束

### 6.1 性能要求

| 指标             | 目标值            |
| ---------------- | ----------------- |
| 冷启动时间       | < 2秒             |
| 内存占用（空闲） | < 200MB           |
| 文件列表加载     | < 500ms（1000项） |
| 大文件上传速度   | 达到带宽上限 90%  |

### 6.2 安全要求

- API Token 必须存储在 OS 密钥链
- 前端不能直接访问 R2 API
- 所有 IPC 调用需要参数验证
- 敏感信息（Secret Key）绝不暴露给前端

### 6.3 S3 API 约束

- 多部分上传分片最小 5MB
- 除最后一个分片外，所有分片必须 ≥ 5MB
- 最多 10000 个分片
- 文件最大 5TB

### 6.4 平台支持

- macOS（ARM64 + x86_64）
- Windows（x86_64）
- Linux（AppImage + deb）
