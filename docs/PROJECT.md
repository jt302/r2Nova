## 🎯 1. 项目愿景

### 1.1 核心定位

**R2Nova** 是「**Cloudflare R2 的 Finder**」。

我们打造一款跨平台的桌面客户端，让开发者能够像操作本地文件管理器一样直观、高效地管理云端对象存储。专注极致性能、极简体验，为中国开发者提供差异化的 R2 管理工具。

### 1.2 一句话描述

> 为中国开发者打造的 Cloudflare R2 专业桌面客户端——50GB 大文件断点续传、多账号无缝切换、类 Finder 的极简操作体验。

### 1.3 使命与价值

| 维度       | 描述                                   |
| ---------- | -------------------------------------- |
| **使命**   | 让对象存储管理像本地文件一样简单自然   |
| **愿景**   | 成为中国开发者首选的 R2 客户端工具     |
| **价值观** | 性能至上、隐私优先、本土优先、开源透明 |

## 👥 2. 目标用户

### 2.1 核心用户画像

#### 用户 A：独立开发者小明

- **背景**: 全栈开发者，个人项目使用 R2 存储静态资源
- **痛点**: 现有工具操作繁琐，大文件上传经常失败
- **需求**: 轻量、快速、支持断点续传、界面中文
- **场景**: 批量上传网站资源、备份项目文件

#### 用户 B：初创团队技术负责人

- **背景**: 10 人团队，使用 R2 作为产品资源存储
- **痛点**: 团队成员共享账号不便，操作记录难追踪
- **需求**: 多账号管理、操作日志、团队协作功能
- **场景**: 团队资源管理、多环境配置切换

#### 用户 C：运维工程师

- **背景**: 负责公司云资源管理，R2 存储业务数据
- **痛点**: 需要频繁查看/导出/清理 Bucket 数据
- **需求**: 快速预览、批量操作、高级筛选、命令行集成
- **场景**: 数据迁移、定期清理、异常排查

### 2.2 用户分层

| 层级 | 用户类型             | 占比 | 优先级   |
| ---- | -------------------- | ---- | -------- |
| P0   | 独立开发者、个人用户 | 60%  | 核心体验 |
| P1   | 中小型团队（<50人）  | 30%  | 协作功能 |
| P2   | 企业用户、运维工程师 | 10%  | 高级功能 |

### 2.3 地域分布

- **中国大陆**: 70%（主要市场，网络优化重点）
- **港澳台/海外华人**: 20%（中文界面需求）
- **其他地区**: 10%（国际化支持）

## 🏆 3. 核心差异化

### 3.1 竞品分析

| 特性           | R2Nova         | Cyberduck | S3Browser | Cloudflare Dashboard |
| -------------- | -------------- | --------- | --------- | -------------------- |
| **安装包体积** | ~5MB           | ~50MB     | ~30MB     | Web（无本地端）      |
| **启动速度**   | < 2秒          | ~5秒      | ~3秒      | 依赖网络             |
| **大文件支持** | 50GB+ 断点续传 | 支持      | 有限      | 浏览器限制           |
| **多账号切换** | 一键切换       | 手动配置  | 手动配置  | 需重新登录           |
| **界面语言**   | 中文/English   | 多语言    | 英文      | 多语言               |
| **离线能力**   | 完全本地       | 完全本地  | 完全本地  | 需联网               |
| **价格**       | 免费开源       | 免费/付费 | 免费      | 免费                 |
| **定制性**     | 开源可定制     | 闭源      | 闭源      | 闭源                 |

### 3.2 核心卖点（USP）

1. **极致轻量** - 5MB 安装包 vs 竞品 50MB+
2. **大文件专家** - 原生支持 50GB+ 断点续传
3. **本土化体验** - 原生中文、国内网络优化
4. **多账号管理** - 无缝切换，团队友好
5. **开源透明** - MIT 协议，社区驱动

### 3.3 护城河

- **性能优势**: Tauri 架构难以被 Electron 竞品超越
- **本土优化**: 针对中国网络环境的深度优化
- **社区生态**: 开源 + 中文社区 = 快速迭代
- **垂直场景**: 专注 R2，不做通用 S3 工具的臃肿功能

## ⚖️ 6. 优先级矩阵

### 6.1 功能优先级（MoSCoW）

**Must Have（必须有）** - 没有这些不能发布

- 账号管理（安全存储、多账号）
- Bucket 文件浏览
- 基础文件操作（上传/下载/删除）
- 断点续传（大文件支持）

**Should Have（应该有）** - 显著提升体验

- 批量操作
- 传输队列管理
- 文件预览（图片、文本）
- 文件搜索
- 暗黑模式

**Could Have（可以有）** - 锦上添花

- 自定义域名分享
- 文件夹树形视图
- 快捷键支持
- 多语言（非中文/英文）

**Won't Have（暂不做）** - 明确排除

- 内置编辑器（使用系统默认应用）
- 文件同步（双向同步复杂度高）
- 实时协作（需要后端）
- 手机端 App（专注桌面端）

### 6.2 技术债务优先级

| 优先级 | 技术债务             | 解决时间      |
| ------ | -------------------- | ------------- |
| P0     | 无测试覆盖           | Sprint 2 之前 |
| P1     | 硬编码配置           | v0.2.0 之前   |
| P2     | 错误处理不完善       | v0.3.0 之前   |
| P3     | 性能未优化（大列表） | v1.0.0 之前   |

## 🔗 8. 依赖与风险

### 8.1 外部依赖

| 依赖              | 类型 | 风险等级 | 备选方案         |
| ----------------- | ---- | -------- | ---------------- |
| Cloudflare R2 API | 必需 | 低       | AWS S3 兼容层    |
| Tauri 2.0         | 必需 | 中       | 降级到 Tauri 1.x |
| aws-sdk-s3        | 必需 | 低       | rust-s3 crate    |
| shadcn/ui         | 高优 | 低       | Radix 原生       |

### 8.2 风险清单

| 风险                     | 概率 | 影响 | 缓解措施                       |
| ------------------------ | ---- | ---- | ------------------------------ |
| Tauri 2.0 稳定性问题     | 中   | 高   | 持续关注 RC 版本，准备降级方案 |
| 大文件传输 Rust 实现复杂 | 中   | 中   | 预留充足时间，参考 S3 官方示例 |
| 国内网络连接 R2 不稳定   | 高   | 中   | 实现智能重试、CDN 加速链接     |
| 竞品推出类似产品         | 低   | 中   | 加快迭代，建立社区壁垒         |
| 开源维护精力不足         | 中   | 高   | 建立核心团队、明确贡献指南     |
